# PROMPT 1

你是一个资深前端工程师 + 视觉设计师。请为歌手「尹毓恪」生成一个可直接部署到 letter.yinyuke.com 的网页项目，用来呈现一封「给粉丝的信」。

1) 项目交付物（必须）
	•	输出一个可运行的前端项目：index.html + styles.css + script.js（或单文件也可，但请明确结构）
	•	不依赖复杂框架（不要 React/Vue），可用原生 JS + CSS
	•	所有资源尽量内置：图形元素使用 SVG / CSS 绘制（不要外链图片；如必须外链，请提供可替换的占位路径）
	•	提供「部署说明」：如何把静态文件部署到域名（例如 Vercel / Cloudflare Pages / Nginx 静态目录）
	•	代码注释清晰，变量命名语义化

2) 视觉与气质（强约束）

整体风格：法国新浪潮电影般的文艺少年感 + 韦斯·安德森式复古配色（柔和蓝彩与冷调）
强调：对称、精致构图、复古手账/信纸质感、旧邮票纹理、打字机/钢笔书写的诗意文字，以及青春躁动感。

请在实现里明确以下视觉细节：
	•	页面像一本「精心制作的复古手账」：纸张纹理、边框、页边距、轻微阴影、细小装饰
	•	构图尽量左右对称，中心聚焦
	•	字体风格：
	•	标题像打字机敲击（可用 CSS 模拟打字机效果，或使用系统字体的类似替代）
	•	正文像钢笔书写/手写感（若不用外链字体，至少用最接近的系统字体，并用 CSS 增强墨迹感）
	•	旧邮票纹理：右上角或边角出现「邮票贴纸」与「邮戳」效果（用 CSS/SVG 生成）
	•	手绘插图元素（必须是你自己生成的 SVG/CSS，不要依赖外部图片）：
	•	春天意象（例如：嫩芽、花瓣、微风线条、燕子）
	•	海啸意象（例如：巨浪线条、浪尖泡沫、潮汐曲线）
	•	两者要以“诗意隐喻”融合，而不是灾难恐怖风

3) 交互流程（必须）
	•	初始状态：信是「未拆封」的状态，屏幕中央出现一个精致信封（对称构图）
	•	用户必须点击/轻触信封，才进入「拆封/打开信」的动画
	•	打开后显示信纸内容（带轻微展开动画）
	•	允许再次合上（可选），但默认打开后停留在阅读状态
	•	动画要克制、电影感、复古：柔和缓动、轻微纸张弹性、轻微颗粒感（可用 CSS 实现）
	•	需要支持移动端与桌面端，自适应布局

4) 内容结构（必须）

信的内容请用占位内容（可写一段短诗/散文式文字），并把以下字段做成可配置常量：
	•	歌手署名（默认：尹毓恪）
	•	信标题
	•	正文段落数组
	•	落款日期
	•	页脚彩蛋（例如一句短句）

5) 「打开次数统计」（必须）

实现“统计这封信被打开了多少次”，定义为：用户点击信封并成功进入已打开状态时计数 +1。
仅做纯前端实现，且不要删除任何现有文件。
